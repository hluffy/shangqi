2     产品简介

车辆定位系统，它主要针对车辆生产车间停放车辆杂乱无章，车辆定位和人员管理等问题而开发的产品，使车辆生产和管理更加科学便捷。

 

2.1      产品特点

本系统使用软硬件结合的方式，定位设备与后台服务器软件结合，将车辆信息和车辆定位信息以及人员信息相结合，一体化显示。可根据多种信息查找车辆位置，定位设备和车辆绑定解绑方便快捷，设备维护简单，可远程配置设置数据，便于管理和运用。

&提示：该系统所有查询都支持模糊查询。

 

2.2      产品功能

车辆的定位系统包括车辆查找、停车区域划分、车辆与定位器绑定、设备信息查询、参数配置、系统管理、设备警报、设备绑定和设备解绑九大模块功能。

该系统登录分两种角色，普通账号和管理员账号。普通用户只有查看功能，包括车辆查找、停车区域划分、车辆与定位器绑定、设备信息查询。管理员账号包括查询功能外还包括参数配置和系统管理。以下对各个功能分别进行介绍。

车辆查找：车辆查找模块包括位置查找、历史轨迹、iBeacon位置三个功能。这些功能主要关注车辆和设备的位置查找，都是在地图上直观的显示具体位置。

1.  位置查找：主要通过缺陷类型、报缺陷人、返修人等条件查询车辆位置，也可通过设备号直接查询车辆位置。

&提示：缺陷类型、报缺陷人、返修人等条件可组合查询，设备号为单独查询条件。

2.  历史轨迹：查看车辆行驶的历史轨迹，通过车架号、开始时间以及结束时间查询车辆在指定的时间里经过的历史轨迹。

3.  iBeacon位置：iBeacon是定位基站，安装在车间的各各角落。可以通过iBeacon编号查找iBeacon基站的安装位置，方便管理。

 

停车区域划分：停车区域划分模块只包含停车区域划分功能。它是将可停车的区域划分成几个区域，点击各个区域可以查看各个区域停车的统计信息，该功能是在地图上直观的显示。

 

车辆与定位器绑定：车辆与定位器绑定模块包含绑定信息功能。显示设备和车辆的绑定和解绑信息，可以通过设备编号或车架号查询。

 

设备信息查询：设备信息查询模块包含LoRa网关信息、iBeacon基站信息、定位设备信息、定位数据信息、停车区域车辆统计五个功能。这些功能主要是对定位设备的配置信息以及对车辆的定位信息进行管理。

1.  LoRa网关信息：显示LoRa网关的配置信息，如LoRa的IP地址、注册服务器端口等信息。可以通过LoRa网关的编号进行查询。

2.  iBeacon基站信息：显示iBeacon基站的基本信息，如电量、安装的经纬度等，通过这些信息可以对iBeacon做后期的维护。可以通过iBeacon编号或电量进行查询。

3.  定位设备信息：显示定位设备的基本信息，如上报时间、电量等。通过电量查询可以了解设备的电量情况，及时更换设备电池，查看上报时间可以了解设备的工作情况。可以通过设备编号查询。

4.  停车区域车辆统计：与停车区域划分功能不同的是，以表格的形式显示各个区域的停车情况。统计出所有区域的停车量以及具体与停车车辆绑定的设备的信息。

 

参数配置：参数配置模块包含定位设备参数设备、LoRa网关参数设置、iBeacon参数设置三个功能。这些功能主要是用于修改各个设备配置数据的修改。

1.  定位设备参数设置：添加或删除定位设备，修改定位设备的配置参数，并同步到设备上去。可以通过设备编号查询。

2.  LoRa网关参数设置：添加或删除LoRa网关，修改LoRa网关的配置参数，并同步到LoRa网关上去。可以通过LoRa网关编号查询。

M注意：在对设备的参数进行修改保存时，系统会主动同步数据。

3.  iBeacon参数设置：添加或删除iBeacon基站，修改iBeacon基站的基本信息。可以通过iBeacon基站编号查询。

&提示：参数配置模块只有管理员账号可见并进行相应操作。

M注意：设备在一开始安装时，所有参数数据都以配置完成。如果没有对设备进行改动，请不要随意修改设备参数。

 

系统管理：系统管理模块包含用户管理和人员配置两个功能。主要是对系统登陆用户和员工基本信息进行管理。

1.  用户管理：显示登录用户的基本信息，添加或删除用户，编辑用户的基本信息。可通过用户名查询。

2.  人员配置：显示员工的基本信息，添加或删除员工，编辑员工的基本信息，也可以excel导入员工信息。可通过员工姓名查询。

&提示：系统管理模块只有管理员账号可见并进行相应操作。

 

设备警报：定位设备和车辆绑定，用于车辆定位 。在车辆经过报交区要回收定位设备，设备是不可以进入物流区的，所以如果设备没有回收则会有报警信息。

 

设备绑定：将设备与车辆信息进行绑定。将定位数据和车辆信息数据关联，在经过161工位 时，通过扫描定位设备号和车架号进行绑定。

&提示：设备绑定功能为单独页面。

 

设备解绑：车辆进入包交区，将设备与车辆解绑，回收定位设备，通过扫描设备号或车架号进行解绑。

&提示：设备解绑功能为单独页面。

3.7      参数配置

参数配置模块是对定位硬件设备进行参数设置，包括定位设备参数设置、LoRa网关参数设置、iBeacon参数设置三个功能。

3.7.1  定位设备参数设置

定位设备参数设置是对车载定位器的基本参数进行设置。静止时间间隔、运行时间间隔、GPS超时时间、LoRa进入休眠时间、有效iBeacon数量、iBeacon超时时间属于定位设备的基本参数。用户可对车载定位器进行添加、编辑、删除、查询等操作。在修改定位设备基本配置参数后可以将数据同步到设备上，达到远程修改参数的目的。

静止时间间隔：是指定位设备在静止时发送两条数据之间的间隔时间。

运行时间间隔：是指定位设备在运行时发送两条数据之间的间隔时间。

GPS超时时间：GPS定位的超时时间，超出时间会重新定位。

LoRa进入休眠时间：是指定位设备接收到LoRa的数据后，休眠的时间。

有效iBeacon数量：定位设备接收到iBeacon的数量小于设置数量时会启动GPS定位，至少为3个。

iBeacon超时时间：iBeacon基站定位的超时时间，超出时间会重新定位。

M注意：设备安装时已经修改好参数，不可随意修改参数。

图3-27

4.1      服务器日常检查和维护

日常安全检测主要针对系统的安全性，工作主要按照以下步骤进行：

1．查看服务器状态：打开进程管理器，查看服务器性能，观察CPU和内存使用状况。查看是否有CPU和内存占用过高等异常情况。

2．查看当前端口开放情况：车辆定位系统必须要开启8080、6262和8090端口。8080是tomcat的启动端口，6262和8090是定位设备的通讯、定位端口。打开命令提示符，输入netstat –ano|findstr “端口号”查看端口是否开启，例如：netstat –ano|findstr “8080”。控制台如下图输出，则表示开启。

图4-1

3．查看数据库运行情况：打开任务管理器中的服务，查看MySQL的状态是否是正在运行。

4．查看tomcat运行情况：打开任务管理器中的服务，查看Tomcat7的状态是否是正在运行。

5．查看相关日志： 主要检查C盘下shangqi文件夹下的shangqilog.txt文件。该文件记录了定位系统运行的日志。

6．查看车辆定位系统各个功能模块运行情况：根据用户手册中第三章操作介绍的内容查看车辆定位系统的各个模块运行是否正常。

4.2      硬件日常检查和维护

这里的硬件包括定位设备、iBeacon基站、LoRa网关三个定位硬件。对定位硬件进行日常检查和维护可以确保定位数据的有效性和准确性，确保服务器可以接受到车辆的定位数据。

 

4.2.1  定位设备

定位设备是与车辆进行绑定的硬件设备。该设备正面编写了设备号和设备号对应的条形码，还有一个指示灯。指示灯会间歇性的连续闪烁。当定位设备在发送数据期间，指示灯会在此期间连续闪烁，当设备发送数据完成后，设备会进入休眠时间，在此期间设备的指示灯会停止闪烁。设备的休眠时间跟设备的工作状态有关。设备在静止状态时，休眠时间是12个小时，设备在运行状态时，休眠时间是10秒。

定位设备的检测一般包括：

1．定位设备的设备号和条形码：查看定位设备的设备号与条形码是否磨损。

2．定位设备的指示灯闪烁：查看定位设备的指示灯是否闪烁。

3．定位设备的外壳：查看定位设备的外壳是否损坏。

解决方法：

对于检测1，如果定位设备的设备号与条形码有磨损并且磨损影响到扫码的数据录入，则需要及时更换设备号与条形码的标签。

对于检测2，如果定位设备的指示灯一段时间未闪烁，则登录到<车辆定位系统>，通过［设备信息查询］列表下的“定位设备信息”链接，在［定位设备信息］界面查询该设备的定位数据。通过表字段［时间］判断设备最后的工作时间，查看设备的电量判断设备是否需要更换电池。

对于检测3，如果定位设备的外壳发生损坏应及时维修更换。

 

4.2.2  iBeacon基站

iBeacon基站安装在车间的各个角落。基站设备正面编写了基站的编号和编号对应的条形码，还有一个指示灯。指示灯会一直连续闪烁。

iBeacon基站的检测一般包括：

1．iBeacon基站安装位置：iBeacon基站安装的位置是否发生变化。

2．iBeacon基站的指示灯闪烁：查看iBeacon基站的指示灯是否闪烁。

3．iBeacon基站的外壳：查看iBeacon基站的外壳是否损坏。

解决方法：

对于检测1，可以通过<车辆定位系统>中［车辆查找］列表下的“iBeacon位置”链接，在［iBeacon位置查找］界面查询到所有或某个iBeacon基站的安装位置。如果iBeacon基站安装位置发生改变，则会影响定位数据的准确性，应该及时将基站复位。

对于检测2，在［设备信息查询］列表下的“iBeacon基站信息”链接，［iBeacon基站信息］界面查询各个iBeacon基站的数据。通过表字段［最后使用时间］查看基站最后的工作时间，查看基站的电量判断基站是否需要更换电池。

对于检测3，如果iBeacon基站的外壳发生损坏应及时维修更换。
